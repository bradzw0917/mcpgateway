# MCP Gateway 多用户远程部署架构

## 问题分析

当 Claude Code 和 Gateway 在不同机器上时：
1. 用户在浏览器中完成阿里云授权
2. 阿里云回调到 Gateway（远程服务器）
3. Gateway 获得了 Token，但无法通知用户本地的 Claude Code
4. Claude Code 一直在等待回调（但回调地址是 localhost，无法到达）

## 解决方案：设备授权流程

```
┌──────────────────┐     ┌──────────────────┐     ┌──────────────────┐
│   Claude Code    │     │   MCP Gateway    │     │     阿里云       │
│   (用户本地)      │     │   (远程服务器)    │     │                  │
└────────┬─────────┘     └────────┬─────────┘     └────────┬─────────┘
         │                        │                        │
         │ 1. 连接 MCP            │                        │
         │ ─────────────────────> │                        │
         │                        │                        │
         │ 2. 返回 401 + 授权URL   │                        │
         │ <───────────────────── │                        │
         │                        │                        │
         │ 3. 用户在浏览器中打开授权URL                    │
         │    ─────────────────────────────────────────>  │
         │                        │                        │
         │                        │ 4. 阿里云回调到 Gateway │
         │                        │ <───────────────────── │
         │                        │                        │
         │                        │ 5. Gateway 存储 Token  │
         │                        │                        │
         │ 6. 轮询 /oauth/device/verify                    │
         │ ─────────────────────> │                        │
         │                        │                        │
         │ 7. 返回授权成功        │                        │
         │ <───────────────────── │                        │
         │                        │                        │
         │ 8. 正常使用 MCP        │                        │
         │ ─────────────────────> │ ─────────────────────> │
```

## 实现要点

1. **设备码机制**：每个授权请求生成唯一的设备码
2. **轮询机制**：Claude Code 轮询 Gateway 检查授权状态
3. **用户标识**：通过设备码关联用户和 Token

## API 设计

### GET /oauth/device/code
生成设备码和授权 URL

### GET /oauth/device/verify?device_code=xxx
检查设备码的授权状态

### POST /oauth/device/complete
Gateway 内部调用，标记授权完成
